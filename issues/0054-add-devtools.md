## 概要

iOS SDK の開発・デバッグを支援する DevTools 機能を作成する。接続状態・統計情報・シグナリングメッセージなどを可視化・操作できる仕組みを提供する。

## 背景

現状は SDK の内部状態やシグナリングの詳細を確認するためにログを読む以外の手段がなく、デバッグ・動作確認に手間がかかる。DevTools として統計情報の可視化や接続パラメーターの確認手段を用意することで、開発効率と問題解決速度を向上させる。

## 実装方針

- SDK のターゲットに組み込むか、別モジュールとして切り出すかを設計時に検討する
- デバッグビルド時のみ有効になる仕組みを優先する
- リリースビルドへの影響がないよう注意する

## 提供候補機能

- `getStats` 結果の構造化・表示（`0048` の RTCRtpCodecCapability ログ出力とも連携）
- シグナリングメッセージのログ表示（送受信の JSON を閲覧できる）
- 接続パラメーター（`Configuration` の内容）の確認
- ICE 接続状態・候補の確認

## 根拠

Sora サーバー側は DevTools を備えており、クライアント側にも同等のデバッグ手段があることで問題の切り分けがしやすくなる。ユーザーからのバグ報告の質向上にも貢献できる。
