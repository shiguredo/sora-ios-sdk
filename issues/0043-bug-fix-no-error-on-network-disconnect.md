# 受信中にネットワークが切断されてもエラー通知がない

## 概要

受信待ち状態中にネットワークが切断された場合、エラーが通知されず接続が成立していない状態のまま待ち続けることがある。

## 再現手順

1. シングルストリーム配信をすでに行っているチャンネルに対し、別のコーデックで受信接続を試みる
2. 次画面に遷移して受信待ち状態になる
3. この状態でネットワークを切断する
4. エラーが通知されずそのまま待ち続ける

## 根拠

ネットワーク切断はアプリが必ず対処すべきイベントであり、エラー通知がなければユーザーに何も伝えられない。接続失敗・切断いずれのケースでも `onDisconnect` または `onFailure` が呼ばれる必要がある。

## 調査方針

- ネットワーク切断時のシグナリングおよび ICE 接続失敗の検出フローを確認する
- `MediaChannelHandlers.onFailure` / `onDisconnect` が呼ばれないパスが存在するか特定する
- ICE の接続状態変化（`iceConnectionState`）から切断を検出する仕組みが正しく機能しているか確認する
